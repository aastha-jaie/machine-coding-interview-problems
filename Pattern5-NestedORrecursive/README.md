This tests your ability to handle data that doesn't have a fixed depth.

The Problems:

1. File Explorer: A tree view where folders can contain files or more folders.

2. Comments Section: A Reddit-style thread with nested replies.

3. Menu/Navbar: Multi-level dropdown menus.

What they check?

 Can you write a #recursive component? 
 Do you understand how to pass state down (or up) through multiple levels of nesting without "Prop Drilling" 
 (e.g., using Context or a flat data structure).
